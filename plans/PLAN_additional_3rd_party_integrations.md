# Technical Specification: Zero-Cost, Unmetered Alert Integration for `saropa-log-capture`

This specification details the implementation plan to integrate five lifetime-free, unmetered alerting vectors into the `saropa-log-capture` VS Code extension workspace. 

---

## 1. Unified Configuration Specification (`package.json`)

To support all five alerting mechanisms natively within the VS Code configuration panel, extend your extension's `package.json` within the `contributes.configuration` block.

```json
{
  "contributes": {
    "configuration": {
      "title": "Saropa Log Capture Alerts",
      "properties": {
        "saropaLogCapture.alerts.enabled": {
          "type": "boolean",
```

## 2. Comprehensive Deep Dive Specs (The 5 Options)

### Option 1: Discord Rich Embed Webhooks
* **Vector Type:** Cloud HTTP REST (Push)
* **Cost Profile:** 100% Free. No billing threshold.
* **Boundaries & Safeguards:** Rate limited at 5 requests per second per webhook. Payloads exceeding 2000 characters in the `content` block or 4096 characters in the `description` block will throw a `400 Bad Request` error. Truncation management is required.

#### User Setup Sequence
1. Navigate to target Discord Server -> **Channel Settings** -> **Integrations** -> **Webhooks**.
2. Click **Create Webhook**, name it "Saropa Logger", and copy the generated Webhook URL.
3. Paste the URL into the `saropaLogCapture.alerts.discordWebhookUrl` extension setting.

#### Concrete Implementation (JavaScript)
```javascript
function dispatchDiscordAlert(webhookUrl, errorName, errorMessage, stackTrace) {
    if (!webhookUrl) return;

    // Hard boundary constraints enforced by Discord API
    const maxStackTraceLength = 1500;
    const sanitizedTrace = stackTrace.length > maxStackTraceLength 
        ? stackTrace.substring(0, maxStackTraceLength) + "\n... [Truncated by Saropa Logger]" 
        : stackTrace;

    const payload = {
        username: "Saropa Log Monitor",
        avatar_url: "[https://marketplace.visualstudio.com/items?itemName=saropa.saropa-log-capture/logo.png](https://marketplace.visualstudio.com/items?itemName=saropa.saropa-log-capture/logo.png)",
        embeds: [{
            title: "🚨 " + errorName,
            description: "**Message:** " + errorMessage,
            color: 14177041, // Red Hex representation
            fields: [
                {
                    name: "Stack Trace Snippet",
                    value: "```javascript\n" + sanitizedTrace + "\n```",
                    inline: false
                }
            ],
            timestamp: new Date().toISOString()
        }]
    };

    fetch(webhookUrl, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify(payload)
    }).catch((err) => console.error("Saropa Discord Alert Error: ", err));
}
```

### Option 2: Telegram Bot API
* **Vector Type:** Cloud HTTP REST (Push)
* **Cost Profile:** 100% Free. No subscription tier exists for API routing.
* **Boundaries & Safeguards:** Rate limits cap at 30 messages per second across all channels. Max character size per message block is 4096 characters using HTML parsing.

#### User Setup Sequence
1. Message `@BotFather` on Telegram, send command `/newbot`, assign names, and copy the provided HTTP API token string.
2. Open a chat window with the newly created bot or add it to a developer group channel, then send a dummy message.
3. Fetch the `chat_id` using `https://api.telegram.org/bot<TOKEN>/getUpdates` and paste both strings into the extension settings block.

#### Concrete Implementation (JavaScript)
```javascript
function dispatchTelegramAlert(botToken, chatId, errorName, errorMessage, stackTrace) {
    if (!botToken || !chatId) return;

    const url = "[https://api.telegram.org/bot](https://api.telegram.org/bot)" + botToken + "/sendMessage";
    
    // Safely escape special HTML control elements
    const cleanName = errorName.replace(/&/g, "&amp;").replace(/</g, "&lt;").replace(/>/g, "&gt;");
    const cleanMessage = errorMessage.replace(/&/g, "&amp;").replace(/</g, "&lt;").replace(/>/g, "&gt;");
    let cleanTrace = stackTrace.replace(/&/g, "&amp;").replace(/</g, "&lt;").replace(/>/g, "&gt;");

    if (cleanTrace.length > 2000) {
        cleanTrace = cleanTrace.substring(0, 2000) + "\n... [Truncated]";
    }

    const htmlText = "<b>🚨 Saropa Log Capture Alert</b>\n\n" +
                     "<b>Type:</b> " + cleanName + "\n" +
                     "<b>Info:</b> " + cleanMessage + "\n\n" +
                     "<b>Stack Trace:</b>\n<code>" + cleanTrace + "</code>";

    const payload = {
        chat_id: chatId,
        text: htmlText,
        parse_mode: "HTML"
    };

    fetch(url, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify(payload)
    }).catch((err) => console.error("Saropa Telegram Alert Error: ", err));
}
```

### Option 3: ntfy.sh (Zero-Auth Pub/Sub)
* **Vector Type:** HTTP PUT/POST Stream
* **Cost Profile:** Completely unmetered on self-hosted architectures. The public instance cloud server handles high volume tiers out-of-the-box completely unauthenticated.
* **Boundaries & Safeguards:** The public platform enforces a global default request size boundary limits at 250 KB per request.

#### User Setup Sequence
1. Invent a completely unique topic string identifier (e.g., `saropa_log_sec_991823`).
2. Subscribe to this phrase via the official iOS/Android/Desktop application layer.
3. Input the topic URL path (`https://ntfy.sh/topic_name`) into the extension settings.

#### Concrete Implementation (JavaScript)
```javascript
function dispatchNtfyAlert(targetUrl, errorName, errorMessage) {
    if (!targetUrl) return;

    // Parameters are passed straight as clean custom Header strings
    fetch(targetUrl, {
        method: "POST",
        body: errorMessage,
        headers: {
            "Title": "Saropa Catch: " + errorName,
            "Priority": "high",
            "Tags": "rotating_light,bug"
        }
    }).catch((err) => console.error("Saropa ntfy Alert Error: ", err));
}
```

### Option 4: macOS Native Core Alerts
* **Vector Type:** Desktop Operating System Runtime Execution (Offline)
* **Cost Profile:** Boundless local execution. Absolute 0 cost.
* **Boundaries & Safeguards:** Requires active execution within Mac OS platforms (`process.platform === 'darwin'`). Text must be fully stripped of single quotes (`'`) to maintain bash parameter formatting inside the AppleScript wrapper shell layer.

#### User Setup Sequence
* **Zero Configuration Required.** The execution engine parses directly into Apple Foundation binaries on the system.

#### Concrete Implementation (JavaScript)
```javascript
const { exec } = require("child_process");

function dispatchMacNativeAlert(errorName, errorMessage) {
    if (process.platform !== "darwin") return;

    // Purge hazardous string injection components breaking shell strings
    const cleanTitle = "Saropa Log Alert".replace(/'/g, "");
    const cleanSubtitle = errorName.replace(/'/g, "");
    const cleanMsg = errorMessage.replace(/'/g, "");

    const appleScriptCmd = "display notification \"" + cleanMsg + 
                          "\" with title \"" + cleanTitle + 
                          "\" subtitle \"" + cleanSubtitle + 
                          "\" sound name \"Glass\"";

    exec("osascript -e '" + appleScriptCmd + "'", (err) => {
        if (err) console.error("Failed to route native macOS notification loop:", err);
    });
}
```

### Option 5: Windows Native Core Alerts
* **Vector Type:** Windows Runtime Execution Subsystem (Offline)
* **Cost Profile:** Boundless local execution. Absolute 0 cost.
* **Boundaries & Safeguards:** Operates inside Windows contexts (`process.platform === 'win32'`). Requires execution bypass flags inside PowerShell commands to handle transient shell scripts safely without breaking execution policy constraints.

#### User Setup Sequence
* **Zero Configuration Required.** Leverages the modern Windows XML Action Center schema natively out of the VS Code node environment.

#### Concrete Implementation (JavaScript)
```javascript
const { spawn } = require("child_process");

function dispatchWindowsNativeAlert(errorName, errorMessage) {
    if (process.platform !== "win32") return;

    // Clean data variables to safely match safe structural XML protocols
    const xmlTitle = "Saropa Error Alert".replace(/&/g, "&amp;").replace(/</g, "&lt;").replace(/>/g, "&gt;").replace(/"/g, "&quot;");
    const xmlSubtitle = errorName.replace(/&/g, "&amp;").replace(/</g, "&lt;").replace(/>/g, "&gt;").replace(/"/g, "&quot;");
    const xmlMsg = errorMessage.replace(/&/g, "&amp;").replace(/</g, "&lt;").replace(/>/g, "&gt;").replace(/"/g, "&quot;");

    const toastXml = "<toast><visual><binding template='ToastGeneric'><text>" + xmlTitle + 
                     " : " + xmlSubtitle + "</text><text>" + xmlMsg + 
                     "</text></binding></visual></toast>";

    // Build optimized execution string targeting Windows Toast Engine assemblies
    const powershellCommand = "[Windows.UI.Notifications.ToastNotificationManager, Windows.UI.Notifications, ContentType = WindowsRuntime] > $null; " +
                              "$xmlDoc = New-Object Windows.Data.Xml.Dom.XmlDocument; " +
                              "$xmlDoc.LoadXml(\"" + toastXml + "\"); " +
                              "[Windows.UI.Notifications.ToastNotificationManager]::CreateToastNotifier('Visual Studio Code').Show($xmlDoc);";

    const child = spawn("powershell.exe", ["-NoProfile", "-ExecutionPolicy", "Bypass", "-Command", powershellCommand]);

    child.on("error", (err) => {
        console.error("Failed to run Windows local notification bridge pipeline:", err);
    });
}
```

## 3. Orchestration Engine (Unified Middleware Router)

Incorporate this orchestration file inside your main logging controller. It automatically reads your user's preferences from the `vscode.workspace` scope and routes payloads to their chosen unmetered pipeline:

```javascript
// alertRouter.js
const vscode = require("vscode");

function routeSaropaLogAlert(errorObject) {
    const config = vscode.workspace.getConfiguration("saropaLogCapture.alerts");
    
    // Master Kill-switch safeguard validation check
    if (!config.get("enabled")) return;

    const activeVector = config.get("selectedVector");
    const errorName = errorObject.name || "Error";
    const errorMessage = errorObject.message || "An unhandled exception occurred.";
    const stackTrace = errorObject.stack || "No call stack discovered.";

    switch (activeVector) {
        case "discord":
            const discordUrl = config.get("discordWebhookUrl");
            dispatchDiscordAlert(discordUrl, errorName, errorMessage, stackTrace);
            break;
        case "telegram":
            const token = config.get("telegramBotToken");
            const cid = config.get("telegramChatId");
            dispatchTelegramAlert(token, cid, errorName, errorMessage, stackTrace);
            break;
        case "ntfy":
            const ntfyUrl = config.get("ntfyTopicUrl");
            dispatchNtfyAlert(ntfyUrl, errorName, errorMessage);
            break;
        case "macosNative":
            dispatchMacNativeAlert(errorName, errorMessage);
            break;
        case "windowsNative":
            dispatchWindowsNativeAlert(errorName, errorMessage);
            break;
        default:
            console.warn("Saropa Alert Router: Matching engine vector index not discovered.");
            break;
    }
}

module.exports = { routeSaropaLogAlert };
```
