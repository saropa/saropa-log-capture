# Performance panel UX and capture master switch

**Summary:** (1) Options panel now has a **Capture** section with **Enable log capture** (master switch for `saropaLogCapture.enabled`). (2) Performance integration is default in `integrations.adapters` so every session gets a snapshot unless the user removes it. (3) Performance icon in the icon bar is shown only when Performance is enabled. (4) Session tab copy clarifies that data cannot be added to an existing log and how to get it next run; overhead note added. (5) Project Logs list shows a performance icon for logs that have performance data; opening such a log shows a **Performance** chip in the session bar that opens the panel. (6) `hasPerformanceData` is derived from session meta and sent in the session list and on load so the viewer can show the chip.

**Files:** Options HTML/script/events, viewer-script-messages, viewer-message-handler, log-viewer-provider-setup, config.ts, package.json (default adapters); viewer-performance-panel (intro, chip), viewer-styles (icon visibility, session list badge, chip); session-history-grouping/metadata, viewer-provider-actions, viewer-session-panel-rendering; log-viewer-provider-load (meta load + setHasPerformanceData with race guard), provider/broadcaster/pop-out setHasPerformanceData; viewer-content (chip), viewer-target.
